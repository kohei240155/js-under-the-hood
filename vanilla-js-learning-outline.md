# JavaScript 深層理解 & フレームワーク内部構造 学習ガイド

> 北米テック企業のフロントエンド面接で、JavaScript の仕組みや React の内部動作を問われたときに自分の言葉で説明できるようになるためのカリキュラム。全セクション手を動かす演習形式。

---

## 1. JavaScript の基礎と実行の仕組み

### 1-1. 型と型変換の罠
- `typeof` で全プリミティブ型（string / number / bigint / boolean / undefined / symbol / null）の結果を確認し、`typeof null === "object"` の罠を実験する
- `==` と `===` で型が異なる値同士を比較する実験を 10 パターン書き、暗黙の型変換がどう起きるかを確認する（例：`"0" == false`, `[] == 0`, `null == undefined`）
- `Number()` / `parseInt()` / `String()` / `Boolean()` の明示的型変換の挙動を一覧表にまとめるコードを書く（空文字列、null、undefined、空配列など境界ケースを含む）

### 1-2. 値渡しと参照渡し
- プリミティブ型の代入でコピーが作られることを実験で確認する
- オブジェクト・配列の代入が参照の共有であることを、片方を変更してもう片方に影響が出る実験で確認する
- シャローコピー（`Object.assign` / スプレッド構文）とディープコピー（`structuredClone`）の違いを、ネストしたオブジェクトで実験する
- 関数の引数にオブジェクトを渡したとき、関数内での変更が呼び出し元に影響する／しないケースをそれぞれ書いて確認する

### 1-3. var / let / const とホイスティング
- `var` / `let` / `const` それぞれの宣言前アクセスを書き、ホイスティングの挙動の違い（undefined vs ReferenceError）を確認する
- 関数宣言と関数式のホイスティングの違いを実験する（`function foo()` は呼び出しより後に書いても動く、`const foo = function()` は動かない）
- ブロックスコープ（`if` / `for` の中）での `var` と `let` の挙動の違いを実験する
- Temporal Dead Zone（TDZ）を意図的に発生させるコードを書いて、エラーメッセージを確認する

### 1-4. コールスタックと実行コンテキスト
- 再帰関数を書いてコールスタックの積み上げを `console.trace()` で可視化する
- 意図的にスタックオーバーフローを起こし、ブラウザの挙動とエラーメッセージを確認する
- ネストした関数呼び出しの実行順序を紙に書いてからコードで検証する

### 1-5. イベントループ・タスクキュー・マイクロタスク
- `setTimeout(0)` / `Promise.resolve().then()` / `queueMicrotask()` / 同期 `console.log` を混在させたコードを書き、実行順序を予測してから実行して答え合わせする
- 上記のパターンを 10 問作って自作クイズ集にする
- `requestAnimationFrame` の実行タイミングを他のタスクと比較する実験コードを書く

### 1-6. 非同期パターンの進化
- Callback だけで 3 つの API を順番に呼ぶコードを書き、ネストの深さ（コールバック地獄）を体感する
- 同じ処理を Promise チェーンで書き直し、コード量と可読性の変化を比較する
- さらに async/await で書き直し、3 パターンを横に並べて差分を観察する
- `Promise.all` / `Promise.race` / `Promise.allSettled` の挙動をそれぞれ実験コードで確認する

### 1-7. エラーハンドリングの仕組み
- `try/catch` で同期エラーと非同期エラー（Promise の reject）の捕捉の違いを実験する
- グローバルの `window.onerror` / `unhandledrejection` イベントでキャッチされないエラーを拾う仕組みを作る
- カスタムエラークラスを `extends Error` で作り、エラーの種類に応じた分岐処理を実装する

### 1-8. メモリ管理とガベージコレクション
- クロージャで意図せず大きなオブジェクトを保持し続けるコードを書き、Chrome DevTools の Memory タブでメモリリークを検出する
- `WeakRef` / `WeakMap` を使ってリークを解消するパターンを実装する
- `setInterval` のクリア忘れによるリークを再現し、修正する

## 2. JavaScript のコア言語仕様

### 2-1. スコープとクロージャ
- `for` ループ + `var` + `setTimeout` のクロージャ問題を実際に書いて「なぜ全部同じ値になるか」を確認する
- `let` で修正するパターンと IIFE で修正するパターンの両方を書いて動作を比較する
- クロージャでプライベート変数を持つカウンターモジュールを作る（`increment()` / `decrement()` / `getCount()` を返す関数）
- カリー化関数 `curry(fn)` を自作して、`add(1)(2)(3)` のような呼び出しを実現する

### 2-2. `this` バインディング
- 同じ関数を 5 つの異なるコンテキストで呼び出すコードを書き、`this` の値をそれぞれ予測 → 検証する（通常呼び出し / メソッド呼び出し / `call` / `new` / アロー関数）
- `Function.prototype.bind` を自作する（`myBind` 関数を書いて、ネイティブの `bind` と同じ挙動になることをテストする）
- React のクラスコンポーネントを書き、`.bind(this)` を外したときに何が壊れるかを実際に確認する

### 2-3. プロトタイプと継承
- `class` を使わずにコンストラクタ関数 + `prototype` で `Animal → Dog` の継承を実装する
- `Object.create` を使った別パターンの継承も実装して比較する
- 同じ継承を `class` / `extends` で書き直し、Babel や TypeScript Playground でトランスパイル結果を確認して「class の裏側でプロトタイプが使われている」ことを目で見る
- `instanceof` の動作をプロトタイプチェーンを手動で辿って再現するコードを書く

### 2-4. 高階関数と関数型パターン
- `Array.prototype.map` を自作する（`for` ループで同じ挙動を再現）
- `Array.prototype.filter` を自作する
- `Array.prototype.reduce` を自作する
- 自作した `map` / `filter` / `reduce` を組み合わせて、商品データの配列から「カテゴリが食品で価格が 500 円以上の商品名を取り出して合計金額を計算する」パイプラインを書く

### 2-5. イテレータとジェネレータ
- `Symbol.iterator` を実装したカスタムイテラブルオブジェクト（例：範囲オブジェクト `range(1, 10)`）を作り、`for...of` で回せるようにする
- ジェネレータ関数で同じ範囲イテレータを書き直し、コードの簡潔さを比較する
- ジェネレータで無限シーケンス（フィボナッチ数列）を作り、必要な分だけ `next()` で取り出す
- async ジェネレータで API のページネーションを実装する（次のページを `for await...of` で逐次取得）

### 2-6. Proxy と Reflect
- `Proxy` でオブジェクトのプロパティアクセスをインターセプトし、存在しないプロパティへのアクセス時にカスタムエラーを投げるバリデーターを作る
- `Proxy` の `set` トラップで、値の変更を自動検知してコンソールにログを出すリアクティブオブジェクトを作る（Vue のリアクティビティの仕組みの簡易版）
- `Reflect` のメソッドを使って、Proxy 内からデフォルトの動作を呼び出すパターンを実装する

### 2-7. モジュールシステム
- 同じユーティリティを CommonJS（`require` / `module.exports`）と ES Modules（`import` / `export`）の両方で書き、Node.js での動作の違いを確認する
- 循環参照が起きるモジュール構成を意図的に作り、CommonJS と ESM でそれぞれどう壊れるかを確認する
- `import()` による動的インポートでコードスプリッティングの効果を DevTools の Network タブで確認する

## 3. DOM・ブラウザ API の低レベル操作

### 3-1. DOM 操作とレンダリングパフォーマンス
- 1000 個の `<li>` を 1 つずつ `appendChild` するコードと `DocumentFragment` でまとめて追加するコードを書き、`performance.now()` で実行時間を計測・比較する
- Chrome DevTools の Performance タブでリフロー・リペイントの発生を実際に確認する
- `offsetHeight` の読み取りが強制リフローを引き起こすことを実験で確認する（読み取りの前後で DOM を変更するパターン）

### 3-2. イベントシステム
- 3 階層にネストした `<div>` にイベントリスナーをつけ、キャプチャリング → ターゲット → バブリングの順序を `console.log` で可視化する
- `stopPropagation()` を各フェーズで呼び出し、伝播がどこで止まるかを確認する
- 動的に追加・削除される Todo リストを作り、親要素へのイベントデリゲーションで全アイテムのクリックを処理する
- React の合成イベント（SyntheticEvent）と同等の仕組みを簡易的に自作する（イベントをルート要素で一括キャッチし、ターゲットに応じてハンドラを振り分ける）

### 3-3. Web API 実践
- `fetch` で JSONPlaceholder API を叩き、ローディング → 成功 → エラーの 3 状態を手動で DOM に反映するコードを書く
- `AbortController` でリクエストキャンセルを実装し、ユーザーが検索欄に連続入力したとき前のリクエストが破棄されることを確認する
- `IntersectionObserver` を使って画像の遅延読み込みを実装する
- `IntersectionObserver` + `fetch` で無限スクロールの商品一覧を実装する

### 3-4. Web Workers
- 重い計算処理（例：巨大配列のソート）をメインスレッドで実行してUIがフリーズすることを確認する
- 同じ処理を Web Worker に移し、UI がブロックされなくなることを確認する
- `postMessage` / `onmessage` によるメインスレッドとワーカー間のデータ通信パターンを実装する

### 3-5. Service Worker と PWA の基礎
- Service Worker を登録し、`fetch` イベントをインターセプトしてオフラインキャッシュを実装する
- キャッシュ戦略（Cache First / Network First / Stale While Revalidate）をそれぞれ実装して動作の違いを確認する
- Web App Manifest を作成し、ホーム画面に追加できる PWA として動かす

## 4. フレームワークの中核概念を自作する

### 4-1. Virtual DOM と差分検出（Reconciliation）
- Virtual DOM ノードを表す JavaScript オブジェクトの構造を設計する（`{ type, props, children }`）
- `createElement(type, props, ...children)` 関数を自作する
- `render(vNode, container)` 関数を自作する（Virtual DOM → 実 DOM の変換）
- `diff(oldTree, newTree)` 関数を自作する（2 つの Virtual DOM ツリーを比較して差分パッチのリストを返す）
- `patch(container, patches)` 関数を自作する（差分パッチを実 DOM に適用する）
- カウンターアプリを作り、状態変更のたびに「全体再描画」と「diff + patch」のパフォーマンスを比較する

### 4-2. リアクティブシステム（useState の自作）
- クロージャで状態を保持する簡易 `useState` 関数を自作する
- 状態変更時に `render` 関数を自動で再実行する仕組みを作る
- 複数の `useState` を同一コンポーネントで使えるようにする（呼び出し順序でインデックス管理 — React の Hooks ルール「条件分岐で呼ぶな」の理由を体感する）
- `useEffect` の簡易版を自作する（依存配列の比較 → クリーンアップ → 副作用実行の流れ）

### 4-3. Fiber アーキテクチャの概念理解
- React が同期レンダリングで長いコンポーネントツリーを処理するとフレーム落ちが起きることを実験で確認する（数千コンポーネントを同期的にレンダリング）
- `requestIdleCallback` を使って作業を小さなチャンクに分割し、ブラウザのアイドル時間に処理する仕組みを実装する
- 4-1 の diff + patch をチャンク分割して実行するように改造し、UI のブロックが解消されることを確認する

### 4-4. コンポーネントシステム
- 関数コンポーネント（状態を受け取って Virtual DOM を返す関数）の仕組みを自作する
- props による親 → 子のデータフローを実装し、逆方向にデータが流れない（単方向データフロー）ことを確認する
- 簡易 `React.memo`（前回の props と今回の props を浅い比較して同じなら再レンダリングをスキップする）を自作する
- 簡易 `Context API` を自作する（Provider でセットした値を、ネストの深い子コンポーネントから直接取得できる仕組み）
- 上記を組み合わせてミニフレームワークとして動く状態にし、Todo アプリを自作フレームワークで作る

### 4-5. クライアントサイドルーター
- `history.pushState` で URL を変更し、`popstate` イベントで画面を切り替える簡易ルーターを自作する
- `/products/:id` のような動的ルートのパラメータ抽出を正規表現で実装する
- 4-4 で作ったコンポーネントシステムと統合し、URL に応じて表示コンポーネントを切り替える
- `import()` による動的インポートでルートごとにコードを分割する実験をする

### 4-6. 状態管理ライブラリの自作
- Pub-Sub パターンで `createStore(reducer)` を自作する（Redux の最小再現）
- `dispatch(action)` → reducer で状態更新 → `subscribe` したリスナーに通知する流れを実装する
- ミドルウェアの仕組み（dispatch をラップして拡張する関数チェーン）を自作する
- 自作ストアを 4-4 のコンポーネントシステムに接続し、グローバル状態の変更でコンポーネントが再レンダリングされることを確認する

## 5. 総合演習：ミニ EC サイトを 3 つのスタックで構築・比較する

### 5-1. Vanilla JS + node:http でフルスクラッチ構築
- フロント：`document.createElement` / `innerHTML` / History API / 手動状態管理で構築する（4章で作ったミニフレームワークは使わず、素の DOM 操作で書く）
- バック：`node:http` のみで API サーバーを構築する（商品一覧 `GET /api/products`、商品詳細 `GET /api/products/:id`、カート CRUD、認証）
- 認証：JWT の生成・検証、Cookie のセット・パースを自前で実装する
- 画面：商品一覧 → 商品詳細 → カート → 注文確認

### 5-2. React + Express で再構築
- フロント：React コンポーネント化、useState / useContext によるカート管理、React Router でルーティング
- バック：Express の Router・ミドルウェア（cors, json パーサー, 認証ミドルウェア）で同じ API を書き直す
- 5-1 と比較して「コード量」「バグの数」「開発速度」を記録する

### 5-3. Next.js で再構築
- ファイルベースルーティング（`app/products/page.tsx`, `app/products/[id]/page.tsx`）
- API Routes（`app/api/cart/route.ts`）によるバックエンド統合
- Server Components / Client Components の使い分け
- 5-2 と比較して「フロントとバックの一体化」による DX 改善を記録する
